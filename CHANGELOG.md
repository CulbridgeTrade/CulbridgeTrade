## Release Process Guidelines
1. **Determine the type of change** (Major, Minor, Patch).
2. **Update the version number** following [Semantic Versioning](https://semver.org).
3. **Document changes** in this CHANGELOG.md file pursuant to the format above.
4. **Create a GitHub release** and tag it with the new version number.
5. **Communicate the release** to stakeholders.
